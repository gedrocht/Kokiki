# Security Policy

## Supported branch

- `main`

## Reporting a vulnerability

Please report vulnerabilities privately through GitHub Security Advisories or
your organization’s approved private security channel.

Do not file public issues for unpatched vulnerabilities.

## Security posture in this repository

- Dependency review on pull requests
- Gitleaks secret scanning
- Trivy filesystem scanning
- CodeQL analysis for Java
- OSSF Scorecard scheduled checks

These safeguards reduce risk, but they do not replace human review.

Note: the dependency review workflow requires GitHub Dependency Graph to be
enabled in the repository settings.
